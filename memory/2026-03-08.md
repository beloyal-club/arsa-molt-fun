## Village Brands Scraper Discovery

**Key finding:** Dutchie uses consistent GraphQL API across ALL sites:
- `FilteredProducts` - product catalog
- `FilteredSpecials` - active sales/promos
- `GetMenuSections` - menu categories

Only the `dispensaryId` changes per site. This pattern works for any Dutchie-powered dispensary.

**Village HBK dispensaryId:** `65eb1d04950e39000920fb22`

**What we can scrape:**
- ✅ Product name, brand, strain type
- ✅ THC/CBD percentages
- ✅ Current price + sale prices
- ✅ In stock status
- ❌ Exact inventory quantity (Dutchie hides this)

## SKU Velocity Architecture (Designed)

Created `docs/architecture/sku-velocity-tracking.md` with:

1. **inventoryEvents table** - track stock state changes (OUT_OF_STOCK, BACK_IN_STOCK)
2. **productVelocity table** - computed daily velocity metrics
3. **Velocity calculation** - count sellouts, avg days in stock
4. **Agent API endpoint** - for me to query velocity insights

**Key insight:** Don't need exact inventory. Track state changes:
- Product goes out of stock frequently = high velocity
- Product stays in stock for weeks = slow mover
